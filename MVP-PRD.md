# Image Background Remover - MVP 需求文档

## 1. 项目概述

### 1.1 产品名称
Image Background Remover - 智能图片背景移除工具

### 1.2 产品定位
一个简单、快速的在线图片背景移除工具，用户上传图片后即可获得透明背景的处理结果。

### 1.3 目标用户
- 电商卖家（快速处理商品图）
- 设计师（快速抠图）
- 普通用户（制作证件照、头像等）

### 1.4 核心价值
- **简单**：无需注册，打开即用
- **快速**：几秒内完成处理
- **免费**：基础功能免费使用

## 2. 功能需求

| 功能 | 描述 | 优先级 |

## 3. 技术架构

### 3.1 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 部署平台 | Cloudflare Workers/Pages | 边缘计算，全球加速 |
| 前端框架 | React + TypeScript | 组件化开发 |
| 后端框架 | Hono | 轻量高效的 Worker 框架 |
| 图片处理 | Remove.bg API | 成熟的 AI 背景移除服务 |
| 存储 | 内存（无持久化） | 图片流式处理，不存储 |

### 3.3 数据流转

```
用户选择图片 
    ↓
前端验证（格式、大小）
    ↓
转换为 base64 / FormData
    ↓
POST /api/remove-bg
    ↓
Worker 转发至 Remove.bg API
    ↓
获取透明背景图片
    ↓
直接返回给前端
    ↓
前端预览 + 提供下载
```

## 4. API 设计

### 4.1 接口列表

| 方法 | 路径 | 描述 |
|------|------|------|
| POST | /api/remove-bg | 移除图片背景 |

### 4.2 详细接口

#### POST /api/remove-bg

**请求**
```http
POST /api/remove-bg
Content-Type: multipart/form-data

image_file: <binary_image_data>
```

**参数**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| image_file | File | 是 | 图片文件，JPG/PNG，最大5MB |

**成功响应**
```http
HTTP/1.1 200 OK
Content-Type: image/png

<binary_png_data>
```

**错误响应**
```http
HTTP/1.1 400 Bad Request
Content-Type: application/json

{
  "error": "FILE_TOO_LARGE",
  "message": "图片大小超过5MB限制"
}
```

**错误码**
| 错误码 | 说明 |
|--------|------|
| FILE_TOO_LARGE | 文件超过5MB |
| INVALID_FORMAT | 不支持的图片格式 |
| PROCESSING_FAILED | 处理失败 |
| API_ERROR | Remove.bg API 错误 |

## 6. 非功能需求

### 6.1 性能指标

| 指标 | 目标 |
|------|------|
| 首屏加载 | < 2s |
| 上传时间 | < 5s (5MB图片) |
| 处理时间 | < 10s |
| 总耗时 | < 20s |

### 6.2 限制

| 项目 | 限制 |
|------|------|
| 单文件大小 | 5MB |
| 支持格式 | JPG, PNG |
| 输出格式 | PNG (透明背景) |
| 并发处理 | 依赖 Remove.bg API 限制 |

### 6.3 安全

- 图片仅内存处理，不持久化存储
- API Key 存储在 Worker 环境变量
- 基础请求频率限制（防滥用）

## 7. 部署配置

### 7.1 环境变量

| 变量名 | 说明 | 必填 |
|--------|------|------|
| REMOVE_BG_API_KEY | Remove.bg API Key | 是 |
| RATE_LIMIT_PER_MINUTE | 每分钟请求限制 | 否，默认 10 |

### 7.2 部署步骤

1. 注册 Cloudflare 账号
2. 安装 Wrangler CLI
3. 配置 REMOVE_BG_API_KEY
4. 运行 `wrangler deploy`
5. 绑定自定义域名（可选）
